1. Executive Summary & System Vision
The Animal Crossing: City Folk RPG Sandbox Map Builder is an immersive, highly interactive, and visually stunning client-server sandbox simulation. It blends the relaxing, cozy aesthetics of Nintendo's classic Animal Crossing: City Folk with a rich, 2D/isometric retro RPG mechanics framework. The application functions as both an interactive map editor and a living RPG simulation. Users can sculpt custom outdoor layouts, manage resources (Bells and Petals), equip specialized gear, and interact with a simulated village of distinct characters.
The application’s central goal is to transition seamlessly across five separate gameplay perspectives (modes of viewing), giving the user diverse, delightful ways to experience their crafted maps. Crucially, the system is backed by an Express backend utilizing the Google Gemini API (via the @google/genai TypeScript SDK). The AI acts as an intelligent co-creator, evaluating town configurations (Isabelle's Feng Shui Audit), facilitating deeply contextual and personality-aligned chat dialogues, and grading custom clothing combinations (Gracie's Fashion Check).
Design Philosophy & Visual Mood
The Cozy Vibrant Palette: Reflective of standard cozy games, the UI features soft pastel greens, warm wood textures, rounded cards, and high-contrast playful typography (pairing "Fredoka" display headings with "Quicksand" for bodies and "JetBrains Mono" for technical values).
Architectural Honesty: No tech-larping, telemetry displays, or simulated terminal logs. All interfaces are presented as intuitive, human-centered elements of a magical town-planning toolkit.
Juice and Feedback: Every interaction—from digging up fossils to watering hybrid roses—triggers visual rewards, subtle screen shakes, floating text indicators, and chronological ledger updates.
Zero-Exposed API Keys: All AI interactions flow through a robust, secure Express proxy server to keep sensitive credentials hidden from client-side bundles.
2. System Architecture & Data Dictionary
The system follows a modern full-stack single-page application (SPA) architecture. The frontend is powered by React 19 and Vite 6, utilizing Tailwind CSS for fluid, fully responsive visual layout. It handles real-time canvas rendering, state tracking, and animations. The backend is an Express 4 application serving as a production-grade asset server and an API gateway for the Gemini model.

// Express Handler (Logical Interface)
import { GoogleGenAI, Modality } from "@google/genai";

export async function handleKKSongRequest(req: any, res: any) {
  const { userPrompt, currentRouteTheme } = req.body;

  const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });

  try {
    // 1. Generate Lyria audio track
    const lyriaResponse = await ai.models.generateContentStream({
      model: "lyria-3-clip-preview",
      contents: `Generate a 30-second cozy acoustic folk track, performed on a single warm nylon-string guitar with a relaxed tempo. Inspired by ${userPrompt} in a ${currentRouteTheme} setting.`,
      config: {
        responseModalities: [Modality.AUDIO],
      }
    });

    let audioBase64 = "";
    for await (const chunk of lyriaResponse) {
      const parts = chunk.candidates?.[0]?.content?.parts;
      if (!parts) continue;
      for (const part of parts) {
        if (part.inlineData?.data) {
          audioBase64 += part.inlineData.data;
        }
      }
    }

    // 2. Generate chord sheet and lyrics via Gemini
    const sheetResponse = await ai.models.generateContent({
      model: "gemini-3.5-flash",
      contents: `Create chords and lyrics for a K.K. Slider song inspired by: ${userPrompt}. Use classic open guitar chords (G, C, D, Em, Am). Use standard Animal Crossing 'Animalese' scat syllables (e.g., 'meow, nah, oh, wah, quee') alongside cute translated english subtitles. Format as structured JSON.`,
      config: {
        responseMimeType: "application/json",
        responseSchema: {
          type: "OBJECT",
          properties: {
            title: { type: "STRING" },
            tempo: { type: "STRING" },
            chordsAndScat: {
              type: "ARRAY",
              items: {
                type: "OBJECT",

7. Engineering Quality: Potential Bug Analysis & Mitigations
Designing complex interactive maps with real-time horizontal scrolling, raycasting, and isometric projection can introduce several frontend and state management bugs. Below is a comprehensive diagnostic plan to identify and mitigate these issues before they impact performance.

  - Missing boundary checks               - Raycaster fish-eye distortion
Bug 1 : Stale Closures in Background Event Handlers & Canvas Listeners
The Scenario: During canvas initialization, the useEffect hook registers event listeners (e.g., mousedown or mouse movement tracking) to detect clicks on Tactical or Isometric tiles.
The Bug: If these event listener callbacks reference active component states (like the selected brushMode, current routes array, or player inventory) directly, they capture stale values from the initial render. Any edit actions made through these listeners will overwrite newer state updates with outdated grid configurations.
The Mitigation: Store the active state values in mutable React refs (brushModeRef, routesRef) updated in lockstep with React state using a synchronization hook:
code
TypeScript
const brushModeRef = useRef(brushMode);
useEffect(() => {
  brushModeRef.current = brushMode;
}, [brushMode]);
Inside the canvas click handler, always reference the updated Ref values (brushModeRef.current) instead of the raw state variables.
Bug 2 : Depth-sorting Visual Artifacts in Isometric Painter's Algorithm
The Scenario: In Isometric view, rendering tall vertical sprites (like cherry blossom trees or standing structures) can cause visual layering issues.
The Bug: If the rendering loop iterates strictly by column-first or row-first order without sorting assets by physical depth, taller background sprites can render on top of shorter foreground items. This breaks the 3D illusion and looks highly unprofessional.

      [Foreground Tree]  <-- Drawn BEFORE background objects?
      [Foreground Rose]  <-- Clipped by overlapping backgrounds!

Correct Depth sorting:
      (Row 0, Col 0) -> (Row 0, Col 1) -> (Row 1, Col 0) -> (Row 1, Col 1)

Bug 3 : Infinite React State Re-renders inside the HTML5 Canvas Draw Loop
The Scenario: The Meadow view runs a continuous requestAnimationFrame loop to render the character's movement animations and update background offsets.
The Bug: If the rendering loop modifies React state variables (like updating bells or push logging) on every frame, it triggers infinite component re-renders. This degrades frame rates, causing visual lag and app crashes.
The Mitigation: Isolate the continuous coordinate variables (like bgOffset and character bounce coordinates) entirely inside mutable refs (bgOffsetRef, characterYRef). Only dispatch React state updates when a discrete game event occurs (such as successfully catching a bug or digging up a fossil).
