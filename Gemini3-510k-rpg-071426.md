Executive Summary & Core Mission1.1 Context and VisionThe medical device sector faces a dual challenge: navigating the strict regulatory hurdles of the U.S. Food and Drug Administration (FDA) Center for Devices and Radiological Health (CDRH) and maintaining reliable software and hardware lifecycles under strict international standards. The Medical Device 510(k) Compliance Simulator (v1.0-Production) is a unified, command-and-control dashboard running in real-time. It transforms traditional gamified concepts into a high-fidelity interactive training platform designed for Regulatory Affairs (RA) professionals, Quality Assurance (QA) engineers, and biomedical reviewers.The simulator puts the user in the role of a Lead FDA CDRH Reviewer or an enterprise RA Director. The primary goal is to evaluate a series of complex medical device submissions—specifically Software as a Medical Device (SaMD) and active electro-mechanical devices—and determine if they are Substantially Equivalent (SE) to a legally marketed predicate device, or if they deserve a Non-Substantial Equivalent (NSE) rejection or a high-stakes Refusal to Accept (RTA) designation.By combining high-performance HTML5 Canvas 2D rendering engines, interactive behavioral micro-activities, and server-side Generative AI orchestrated via the @google/genai SDK and Gemini models, this application provides users with realistic regulatory pressure, dynamic risk mitigation scenarios, and automated design-control validation systems.1.2 Core CapabilitiesClinical Diagnostic Viewport (Multiplexer Component): A multi-mode spatial display showing dynamic biosignals, 3D wireframe anatomical tracking overlays, historical device comparative component trees, and real-time medical imaging simulations.Regulatory Micro-Activities (Interactive Core): Hands-on operational training systems that simulate software lifecycle tracking (SDLC Code Vulnerability Runner), real-time clinical adverse event interception (Adverse Event Triage Smasher), and wireless signal protection validation (Cybersecurity Telemetry Signal Tuner).CDRH Expert AI Cognitive Cores: A server-side intelligence network specifications, hazard classification levels, and original design logs for both the subject and predicate devices.2. To ensure rapid execution across containerized runtime environments like Google Cloud Run or AWS Fargate, and to bypass potential ES Module path resolution errors, the system relies on an integrated esbuild compilation workflow:Bashvite build && esbuild server.ts --bundle --platform=node --format=cjs --packages=external --sourcemap --outfile=dist/server.cjs
Selection Rationale for CommonJS (.cjs) BundlingPath Resolution Simplification: Native Node.js ES Modules require explicit filename extensions on local file imports. Bundling to a unified CommonJS architecture handles these local relative paths at build time, eliminating runtime file lookup errors.External Library Exclusion: Utilizing the --packages=external flag instructs esbuild to skip heavy node modules (such as express or @google/genai) during code bundling. Instead, these are resolved directly by the container's environment, keeping production deployment weights low.Container Cold-Start Optimization: Consolidating backend infrastructure into a unified file (dist/server.cjs) eliminates filesystem I/O overhead when scaling out containers, which accelerates initial request responses.3. The system uses a native browser Web Audio API oscillator synthesizer:TypeScriptclass ComplianceSynth {
  private ctx: AudioContext | null = null;

  private init() {
    if (!this.ctx) {
      this.ctx = new (window.AudioContext || (window as any).webkitAudioContext)();
    }
  }

  public playTelemetryClick() {
    this.init();
    if (!this.ctx) return;
    
    const osc = this.ctx.createOscillator();
    const gain = this.ctx.createGain();
    
    osc.type = "sine";
    osc.frequency.setValueAtTime(1200, this.ctx.currentTime);
    osc.frequency.exponentialRampToValueAtTime(300, this.ctx.currentTime + 0.05);
    
    gain.gain.setValueAtTime(0.02, this.ctx.currentTime);
    gain.gain.exponentialRampToValueAtTime(0.001, this.ctx.currentTime + 0.05);
    
    osc.connect(gain);
    gain.connect(this.ctx.destination);
    
    osc.start();
    osc.stop(this.ctx.currentTime + 0.05);
  }

  public playAlertWarning() {
    this.init();
    if (!this.ctx) return;
    
    const osc = this.ctx.createOscillator();
    const gain = this.ctx.createGain();
    
    osc.type = "sawtooth";
    osc.frequency.setValueAtTime(220, this.ctx.currentTime);
    osc.frequency.exponentialRampToValueAtTime(50, this.ctx.currentTime + 0.4);
    
    gain.gain.setValueAtTime(0.15, this.ctx.currentTime);
    gain.gain.exponentialRampToValueAtTime(0.001, this.ctx.currentTime + 0.4);
    
    osc.connect(gain);
    gain.connect(this.ctx.destination);
    
    osc.start();
    osc.stop(this.ctx.currentTime + 0.4);
  }
}

  const sanitized = rawText.trim();
  if (!sanitized) return fallbackObject;

  try {
    // Attempt standard direct parse
    return JSON.parse(sanitized) as T;
  } catch (initialError) {
    try {
      // Use regex to extract JSON content from inside markdown blocks
      const jsonRegex = /<\/s>|^```json\s*([\s\S]*?)\s*```/m;
      const match = jsonRegex.exec(sanitized);
      
      if (match && match[1]) {
        return JSON.parse(match[1].trim()) as T;
      }
      
      // Secondary attempt: isolate content between the first and last brace
      const firstBrace = sanitized.indexOf('{');
      const lastBrace = sanitized.lastIndexOf('}');
      
      if (firstBrace !== -1 && lastBrace !== -1) {
        const structuralSlice = sanitized.slice(firstBrace, lastBrace + 1);
        return JSON.parse(structuralSlice) as T;
      }
    } catch (nestedError) {
      console.error("AI JSON extraction failed. Utilizing fallback layout.", nestedError);
    }
    
    return fallbackObject;
  }
}
